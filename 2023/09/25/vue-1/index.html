vue.js 学习笔记

  • 这种命名方法能显而易见的表明这是一个组件
  • idea/vscode等编译器方便进行识别

组件

组件实例化

组件的本质就是一个个的构造函数，这些函数以组件定义的options对象为参数。 最顶级的组件就是从vue.js中导入的Vue函数。

在这个过程中我们主要干了哪几件事情：

  • 使用new 关键字实例化一个对象，vue是一个类。在js中类用Function 定义

  • 在Vue（） 函数中调用初始化函数， vue初始化主要干了几件事情

    • 初始化内部的数据结构。
    • 设置 _uid (唯一标识符)。
    • 初始化生命周期 (lifecycle)。
    • 初始化事件 (events)。
    • 初始化渲染函数 (render function)。
    • 初始化数据 (data) 和属性 (props)。
    • 初始化方法 (methods)。
    • 初始化计算属性 (computed properties)。
    • 初始化观察者 (watchers)。

  • 编译：

    • 将模板字符串编译为dom元素
    • 解析模板中的指令和插槽

  • 挂载

    将虚拟dom挂载到真实的dom上

    触发响应的生命周期钩子

new 一个对象的过程：

生命周期：

接下来说明一下每个生命周期的含义：

  • beforeCreate
    在组件实例初始化之初，在生命周期，事件系统，组件渲染初始化之后

    官网说明：

    查看源码发现这个钩子函数的调用是:

  • created
    实例已经创建完成，props,data,methods,computed,watch 等已经初始化完毕，但是还没有挂载到 DOM 上，也没有渲染模板。

  • beforeMount

    在挂载开始之前被调用：相关的 render 函数首次被调用。

  • mouted

    el 被新创建的 vm.$el 替换，并挂载到实例上去之后调用该钩子。如果 root 实例挂载了一个文档内元素，当 mounted 被调用时 vm.$el 也在文档内。

  • beforeUpdate

    数据更新时调用，发生在虚拟 DOM 重新渲染和打补丁之前。这里适合在更新之前访问现有的 DOM，比如手动移除已添加的事件监听器。

  • updated

    由于数据更改导致的虚拟 DOM 重新渲染和打补丁，在这之后会调用该钩子。当这个钩子被调用时，组件 DOM 已经是更新后的状态了。但是你应该避免在此期间更改状态，因为这可能会导致无限循环的更新。

  • beforeDestroy

    实例销毁之前调用。在这一步，实例仍然完全可用

  • destroyed

    Vue 实例销毁后调用。在这一步，所有的事件监听器会被移除，所有的子实例也会被销毁。

  • actived

    如果页面有 缓存机制，当组件被激活时调用

  • deactived

    如果页面有 缓存机制，当组件被停用时调用

  • errorCaptured

    当捕获一个来自子孙组件的错误时被调用

  • severPrefetch

    仅在服务端渲染时调用，用于提前获取异步数据。

  • renderTracked

    当任何响应式依赖更新时调用。

  • renderTrikked

为什么建议将数据请求放在created当中，而不是mouted当中

created是组件实例化后立即调用。这时候页面dom节点未生成。mouted调用时dom节点已经生成。可能会导致页面闪动

首先， provide和inject的初始化是在beforeCreate函数调用之后去初始化的.。

双向数据绑定

主要查看vue源码中的core中的observer部分（core/observer）

大致流程

  • new vue初始化的时候， 对data执行相应化处理。这个过程发生在observer中
  • 同时对模板进行编译，找到其中动态绑定的数据。 从data中获取并初始化视图，这个过程发生在Compile中
  • 同时定义一个更新函数和watcher。数据每次更新时，watcher会调用灯芯函数
  • data中的key在某个视图中可能会出现多次。所以有一个管家dep来管理多个watcher
  • 一旦数据发生变化，就会找到对应的watch。通知所有watcher执行更新函数

服务端渲染(SSR)

什么叫做渲染： 就是数据和图片填充到dom元素中

服务端渲染： 服务端拿出数据来，直接放到dom当中

  • 比如在服务端返回的js代码中。写一段php代码去解析数据，返回解析好的html文件就是服务端渲染

    • 爬虫爬的时候能直接爬到数据，能够提高seo，博客。可以使用这种方式
    • 渲染更快，减少请求速度

    缺点

    • 前后端一起开发， 前后端耦合性较强
    • 前端的一些框架无法使用
      • react框架可以替换成next.js, remix，faas，基于vue的nuxt等

客户端/浏览器渲染： 向服务端请求数据，然后再浏览器或者客户端做填充
